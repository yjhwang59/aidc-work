# 品牌指南

> AIDC.work 視覺、語氣與溝通規範

## 1. 品牌核心

### 1.1 品牌名稱

**AIDC.work**

- **AIDC** = AI Development Consultant（企業 AI 發展顧問）
- **.work** = 強調實際工作成果與可落地的專業產出

### 1.2 品牌主理人

**Jack Y. J. Hwang**（黃彥傑）

企業 AI 發展顧問，專注於協助企業將 AI 工具、Vibe Coding 與 AI Agent 轉化為可運行的系統與工作流。

### 1.3 定位語

**中文**：

> 協助企業從 AI 工具使用，走向 AI 系統開發與 AI Agent 工作流落地。

**英文**：

> Helping enterprises move from AI tool usage to AI system development and AI Agent workflow implementation.

### 1.4 品牌角色

| 角色 | 說明 |
|------|------|
| 顧問 | 提供策略、方法論與落地陪跑 |
| 教育者 | 透過課程與文章傳遞實務知識 |
| 建構者 | 展示 AI 開發與 Agent 工作流的實際成果 |
| 資產管理者 | 長期累積可複用的模板、案例與工作流 |

## 2. 品牌個性

### 2.1 個性關鍵字

| 特質 | 表現方式 |
|------|----------|
| 專業 Professional | 用數據與案例說話，避免空泛承諾 |
| 務實 Practical | 聚焦可執行的步驟與可複製的方法 |
| 可信賴 Trustworthy | 誠實說明限制與風險，不過度包裝 |
| 清晰 Clear | 結構分明，少 jargon、多例子 |
| 顧問感 Consultant-like | 像陪跑夥伴，不是推銷員 |

### 2.2 避免的形象

- ❌ 過度炫技或誇大 AI 能力
- ❌ 堆砌流行語（「顛覆」「革命」「萬能」）
- ❌ 過度商業化的推銷語氣
- ❌ 缺乏深度的清單式內容農場文章
- ❌ 視覺過於花俏、缺乏專業感

## 3. 語氣與文案

### 3.1 語氣原則

| 情境 | 語氣 | 範例 |
|------|------|------|
| 首頁 Hero | 自信、簡潔 | 「協助企業將 AI 從實驗走向落地。」 |
| 服務說明 | 專業、具體 | 「透過 Vibe Coding 工作坊，讓團隊在兩天內完成可運行的原型。」 |
| 部落格 | 教學、洞察 | 「多數企業在導入 AI Agent 時，忽略了權限與稽核設計。」 |
| 聯絡 CTA | 友善、明確 | 「想討論您的 AI 導入計畫？歡迎與我聯繫。」 |

### 3.2 用語規範

**建議使用**：

- 落地、導入、工作流、系統化、陪跑、評估、路線圖
- AI 工具、Vibe Coding、AI Agent、企業培訓

**謹慎使用**：

- 革命性、顛覆、萬能、保證成功
- 過度絕對的承諾（「一定」「100%」）

### 3.3 語言

- **主要語言**：繁體中文
- **次要語言**：英文可用於技術名詞、專業職稱、雙語標題
- 首次出現的英文術語，可附括號中文說明

## 4. 視覺方向（MVP 指引）

> MVP 階段以 Tailwind CSS 實作，以下為設計方向指引，非最終設計規格。

### 4.1 整體風格

- 簡潔、專業、留白充足
- 以內容為主角，減少裝飾性動畫
- 響應式設計，行動裝置優先可讀性

### 4.2 色彩方向（建議）

| 用途 | 方向 |
|------|------|
| 主色 | 深藍或墨藍 — 傳達專業與信任 |
| 輔色 | 中性灰階 — 背景、邊框、次要文字 |
| 強調色 | 單一亮色（如青綠或琥珀）— CTA 按鈕、連結 |
| 背景 | 淺色為主，可支援深色模式（Post-MVP） |

### 4.3 字體方向（建議）

| 用途 | 方向 |
|------|------|
| 中文內文 | 系統無襯線字體或 Noto Sans TC |
| 英文 / 程式碼 | 系統無襯線或等寬字體（程式碼區塊） |
| 標題 | 與內文同族，透過字重與大小區分層級 |

### 4.4 圖像與圖示

- 優先使用簡潔的線條圖示
- 架構圖、流程圖優先於裝飾性插畫
- 案例與課程可搭配示意圖或截圖
- 避免過度 AI 生成的通用商業人物圖

## 5. 標誌與識別

### 5.1 Logo 使用（待設計）

- 文字標誌：**AIDC.work** 或 **AIDC**
- 使用場景：Header、Footer、社群頭像、favicon
- 保持足夠留白，不擠壓變形

### 5.2 命名一致性

| 正確 | 避免 |
|------|------|
| AIDC.work | aidc、AIDC Work、aidc-work（對外品牌） |
| Jack Y. J. Hwang | 小寫 jack、不一致的中英文姓名格式 |
| Vibe Coding | vibe coding（正文首次出現除外） |
| AI Agent | ai agent、智能代理（除非刻意使用中文） |

## 6. 內容中的品牌呈現

### 6.1 頁面元素

| 元素 | 規範 |
|------|------|
| 頁面標題 | `[頁面名稱] — AIDC.work` |
| Meta 描述 | 一句話說明頁面價值，含關鍵服務詞 |
| 作者署名 | Jack Y. J. Hwang |
| 版權 | © [年份] Jack Y. J. Hwang. All rights reserved. |

### 6.2 文章與案例

- 開頭或結尾可附簡短顧問署名與 CTA
- 案例需標註是否為匿名處理
- 模板需說明授權與使用方式

## 7. 對外溝通場景

| 場景 | 品牌重點 |
|------|----------|
| 網站首頁 | 定位語 + 服務摘要 + 信任建立（案例） |
| 社群分享 | 文章標題 + 一個具體洞察 + 連結 |
| 合作提案 | 強調方法論、陪跑模式、可量化成果 |
| 課程宣傳 | 學習目標明確、適合對象清楚 |

## 8. 品牌檢核表

內容或設計發布前確認：

- [ ] 語氣專業、務實，無過度推銷
- [ ] 使用繁體中文為主
- [ ] 技術名詞拼寫一致
- [ ] 視覺簡潔，不搶內容風采
- [ ] CTA 明確但不咄咄逼人
- [ ] 品牌名稱與主理人署名正確
