# Issue 模板

> 複製以下模板至 GitHub / GitLab Issue 建立頁面使用

---

## 功能需求 Issue

```markdown
## 類型
- [ ] 新功能
- [ ] 內容新增
- [ ] 錯誤回報
- [ ] 改善建議
- [ ] 文件更新

## 摘要
<!-- 用一句話描述這個 Issue 要解決什麼 -->

## 背景與動機
<!-- 為什麼需要這個變更？對應哪個產品目標或使用者需求？ -->

## 預期行為
<!-- 完成後應該是什麼樣子？ -->

## 相關頁面 / 路由
<!-- 例如：/blog, /cases/[slug], 首頁 Hero 區塊 -->

## 驗收條件
- [ ] 條件一
- [ ] 條件二
- [ ] 條件三

## 補充資訊
<!-- 參考連結、截圖、相關 Issue -->

## 優先級
- [ ] 高（阻擋上線或核心功能）
- [ ] 中（改善體驗或內容品質）
- [ ] 低（可延後處理）
```

---

## 內容新增 Issue

```markdown
## 內容類型
- [ ] 部落格文章
- [ ] 案例研究
- [ ] 課程說明
- [ ] 模板與工具

## 標題
<!-- 建議的文章或資源標題 -->

## 內容支柱
- [ ] 企業 AI 導入與策略
- [ ] AI 工具與 Vibe Coding
- [ ] AI 系統與 Agent 工作流
- [ ] 顧問模板與實務工具

## 目標讀者
<!-- 例如：企業決策者、技術主管、開發團隊 -->

## 摘要
<!-- 2–3 句話說明這篇內容的價值 -->

## 建議 slug
<!-- 例如：ai-agent-integration-pitfalls -->

## 大綱
1. 
2. 
3. 

## 驗收條件
- [ ] frontmatter 欄位完整
- [ ] 繁體中文正文
- [ ] 符合品牌語氣（參考 docs/brand-guide.md）
- [ ] 本機預覽渲染正確
- [ ] 無未授權客戶資訊

## 相關文件
- docs/content-strategy.md
- docs/brand-guide.md
```

---

## 錯誤回報 Issue

```markdown
## 問題摘要
<!-- 一句話描述問題 -->

## 重現步驟
1. 
2. 
3. 

## 預期行為
<!-- 應該發生什麼？ -->

## 實際行為
<!-- 實際發生什麼？ -->

## 環境
- 瀏覽器：
- 裝置：（桌機 / 手機 / 平板）
- 頁面 URL：
- 分支 / 版本：（若已知）

## 截圖或錯誤訊息
<!-- 附上截圖或 console 錯誤 -->

## 嚴重程度
- [ ] 阻擋使用
- [ ] 功能異常但可繞過
- [ ] 輕微 UI 問題
```

---

## 改善建議 Issue

```markdown
## 改善項目
<!-- 想改善什麼？ -->

## 現況
<!-- 目前的做法或體驗 -->

## 建議方案
<!-- 你的改善想法 -->

## 預期效益
<!-- 改善後對使用者或維運有什麼幫助？ -->

## 是否願意貢獻 PR？
- [ ] 是
- [ ] 否，僅提供建議
```
