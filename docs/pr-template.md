# Pull Request 模板

> 建立 PR 時複製以下模板至描述欄位

---

## 標準 PR 模板

```markdown
## 摘要
<!-- 1–2 句話說明這個 PR 的目的與價值 -->

## 變更類型
- [ ] 新功能（feat）
- [ ] 錯誤修復（fix）
- [ ] 內容新增或更新（content）
- [ ] 樣式調整（style）
- [ ] 重構（refactor）
- [ ] 文件（docs）
- [ ] 其他（chore）

## 相關 Issue
<!-- Closes #123 或 Relates to #456 -->

## 變更說明
<!-- 列出主要變更項目 -->
-
-

## 影響範圍
<!-- 哪些頁面、路由或功能受影響？ -->
- 

## 測試方式
<!-- 審閱者或自己如何驗證？ -->
- [ ] `npm run build` 通過
- [ ] 本機 `npm run dev` 預覽正常
- [ ] 響應式版面確認
- [ ] 連結與內容渲染正確

## 截圖（若為 UI 變更）
<!-- 桌機 / 手機截圖 -->

## 檢核清單
- [ ] 遵循專案慣例與 AGENTS.md 指引
- [ ] 未引入非必要依賴
- [ ] 內容使用繁體中文，符合品牌語氣
- [ ] 無敏感資訊或未授權客戶資料
- [ ] 變更範圍聚焦，無無關修改
```

---

## 內容 PR 模板

適用於僅新增或更新 `content/` 目錄的 PR。

```markdown
## 摘要
<!-- 新增哪篇內容？主題與目標讀者是誰？ -->

## 內容類型
- [ ] 部落格文章
- [ ] 案例研究
- [ ] 課程說明
- [ ] 模板與工具

## 新增檔案
<!-- 列出 content/ 下的新檔案 -->
- `content/blog/example-slug.mdx`

## 內容支柱
- [ ] 企業 AI 導入與策略
- [ ] AI 工具與 Vibe Coding
- [ ] AI 系統與 Agent 工作流
- [ ] 顧問模板與實務工具

## 品質檢核
- [ ] frontmatter 完整（title, description, date, tags）
- [ ] 繁體中文正文
- [ ] 符合 docs/brand-guide.md 語氣規範
- [ ] 客戶資訊已匿名或取得授權
- [ ] slug 使用 kebab-case
- [ ] 本機預覽渲染正確

## 相關 Issue
<!-- Closes #123 -->
```

## PR 標題建議格式

```text
feat: 新增部落格列表頁與動態路由
fix: 修正案例頁日期顯示格式
content: 新增 AI Agent 整合常見陷阱文章
docs: 建立專案基礎文件
style: 調整首頁服務卡片間距
```
