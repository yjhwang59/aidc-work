# 產品需求文件（PRD）

> AIDC.work — Jack Y. J. Hwang 企業 AI 發展顧問官方網站

## 1. 產品概述

### 1.1 產品名稱

**AIDC.work**（Enterprise AI Development Consultant）

### 1.2 產品定位

AIDC.work 是 Jack Y. J. Hwang 的專業顧問網站，同時扮演三種角色：

| 角色 | 說明 |
|------|------|
| **企業 AI 發展顧問網站** | 向企業決策者與技術團隊展示顧問服務、方法論與合作方式 |
| **顧問內容資產庫** | 長期累積文章、案例、模板、工作流與 AI 輔助開發範例 |
| **AI 開發成果展示平台** | 呈現 Vibe Coding、AI Agent 工作流與實際系統落地成果 |

### 1.3 核心價值主張

> 協助企業從 AI 工具使用，走向 AI 系統開發與 AI Agent 工作流落地。

### 1.4 目標使用者

| 使用者類型 | 需求 |
|------------|------|
| 企業決策者（CXO、部門主管） | 了解 AI 導入路線圖、顧問服務範圍、成功案例 |
| 技術負責人（CTO、IT 主管、架構師） | 評估技術可行性、工作流設計、Agent 落地方式 |
| 開發與產品團隊 | 參考模板、課程、實作範例與最佳實務 |
| 潛在合作夥伴 | 了解專業背景、服務項目與聯絡方式 |

## 2. 產品目標

### 2.1 MVP 目標

- 建立可信賴的顧問形象與服務說明
- 提供可瀏覽的內容資產（文章、案例、模板、課程）
- 以靜態優先架構快速上線，降低維運成本
- 作為長期內容累積的基礎平台

### 2.2 非目標（MVP 階段不做）

- 使用者帳號與登入系統
- 後台 CMS 或資料庫
- 線上付款或課程報名系統
- 即時聊天或客服機器人
- 多語系完整切換（技術名詞可保留英文）

## 3. 功能需求

### 3.1 頁面與路由

| 路由 | 頁面名稱 | 主要功能 |
|------|----------|----------|
| `/` | 首頁 | 品牌定位、核心服務摘要、精選內容、行動呼籲 |
| `/about` | 關於我 | 顧問背景、專業領域、合作理念 |
| `/services` | 服務項目 | AI 工具培訓、Vibe Coding、系統開發、Agent 工作流、導入路線圖 |
| `/cases` | 案例研究 | 企業 AI 導入與開發成果案例 |
| `/courses` | 課程與培訓 | 培訓主題、形式說明、學習目標 |
| `/blog` | 部落格 | 顧問觀點、技術文章、導入心得 |
| `/templates` | 模板與工具 | 可下載或參考的顧問模板、工作流、Prompt 範例 |
| `/contact` | 聯絡我們 | 合作洽詢表單或聯絡資訊 |

### 3.2 內容管理

內容以 Markdown / MDX 檔案存放於 `content/` 目錄：

```text
content/
├─ blog/        # 部落格文章
├─ cases/       # 案例研究
├─ courses/     # 課程說明
└─ templates/   # 模板與工具資源
```

每篇內容應包含 frontmatter 中繼資料（標題、摘要、日期、標籤、分類等）。

### 3.3 內容類型

| 類型 | 說明 | 範例 |
|------|------|------|
| 文章 | 顧問觀點與技術分享 | 「企業導入 AI Agent 的三個常見陷阱」 |
| 案例 | 匿名或授權的實際導入故事 | 「製造業 AI 工作流自動化案例」 |
| 模板 | 可複製的工作文件與流程 | 「AI 導入評估檢核表」 |
| 課程 | 培訓主題與學習大綱 | 「Vibe Coding 實戰工作坊」 |

### 3.4 通用功能

- 響應式版面（桌機、平板、手機）
- 內容列表與詳情頁
- 標籤或分類篩選（可依實作階段逐步加入）
- SEO 友善的頁面標題與描述
- 清晰的導覽列與頁尾

## 4. 非功能需求

### 4.1 技術架構

- **框架**：Next.js App Router
- **語言**：TypeScript
- **樣式**：Tailwind CSS
- **內容**：MDX / Markdown
- **部署**：靜態優先（Static-first）
- **依賴**：不引入非必要套件

### 4.2 效能

- 首屏載入時間目標 < 3 秒（一般網路環境）
- 靜態頁面預渲染，減少伺服器負載
- 圖片適當壓縮與延遲載入

### 4.3 可維護性

- 內容與程式碼分離
- 遵循專案慣例與 `docs/` 文件規範
- 變更透過 Git 版本控制與 PR 流程

### 4.4 品牌一致性

- 語氣專業、務實、可信賴
- 避免過度炫技或堆砌流行語
- 主要語言為繁體中文，技術名詞可保留英文

## 5. 使用者旅程

```text
1. 透過搜尋或推薦進入首頁
2. 瀏覽服務項目與案例，建立信任
3. 閱讀部落格或下載模板，獲取實用價值
4. 查看課程與培訓選項
5. 透過聯絡頁面發起合作洽詢
```

## 6. 成功指標（MVP）

| 指標 | 說明 |
|------|------|
| 網站上線 | MVP 八個主要路由可正常瀏覽 |
| 內容基礎量 | 至少各類型有 1–2 篇種子內容 |
| 聯絡轉換 | 聯絡頁面資訊完整、表單或管道可用 |
| 維運效率 | 新增一篇文章不需改動核心程式碼 |

## 7. 未來擴充（Post-MVP）

以下功能不在 MVP 範圍，但可在產品路線圖中規劃：

- Newsletter 訂閱
- 內容搜尋與進階篩選
- 英文版或雙語切換
- 線上課程報名整合
- 訪客分析與轉換追蹤
- 輕量 CMS 或 headless CMS 整合
